API Backend Project: URL Shortener Microservice

User stories:

  1. I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
  2. If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
  3. When I visit that shortened URL, it will redirect me to my original link.

Short URL Create Examples

https:/???.herokuapp.com/new/https://www.google.com
https://???.herokuapp.com/new/http://foo.com:80

When a short URL created successfully, it should return back this example output:

{ "original_url": "http://foo.com:80", "short_url": "https://???.herokuapp.com/8170" }

Short URL use example

Simply use short URL returned by this microservice in the browser, for example:

https://???.herokuapp.com/2871

and it should redirect to the original URL:

https://www.google.com/

Short URL Create Example with Invalid URL

When a user tries to create short URL for invalid URL, the service will return en error. For example:

https:/???.herokuapp.com/new/https://www

will return:

{ "error": "URL Invalid" }
